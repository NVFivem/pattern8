# Pattern 8 (P8) — Agent 行为准则

你正在一个受 Pattern 8 (P8) 框架治理的项目中工作。你绝对必须遵守以下所有规则。

## 核心约束

在开始处理任何任务之前，你必须：

1. **检查 `skills/` (或 `skills_zh/`) 目录**，找到与当前任务最匹配的 SKILL。
2. **仔细阅读该 SKILL 下的所有文件** (`SKILL.md` + `assets/` + `references/`)。
3. **严格遵守 `SKILL.md` 中定义的流水线步骤**。

## 五大守则 (严禁跳过)

### 1. 流水线 (Pipeline)
所有任务必须严格按照所在 `SKILL.md` 定义的顺序执行。绝对不允许跳过任何一个步骤。

### 2. 反转模式 (Inversion)
在正式干活之前，必须逐一核对 `assets/checklist.yaml` 里的清单。
如果用户提供的需求缺少前置依赖或关键信息，你必须**立即停下来并向用户提问**，绝对不允许靠幻觉或擅自猜测来填补空缺。

### 3. 生成器强制 (Generator)
你的输出格式必须完全符合 `assets/template.yaml` 中定义的框架。
不允许自由发挥，模板中的每一个字段/标题都必须被填充。

### 4. 工具使用围栏 (Tool Wrapper)
在执行任何终端命令或修改文件前，必须先查阅 `references/security.yaml`。
凡是命中黑名单或者违反红线安全规则的操作，必须立刻被丢弃。

### 5. 自我审查 (Reviewer)
在生成完代码或文档后，必须拿着 `references/guidelines.yaml` 中的标准对自己的输出进行一次自审。
如果不合规，退回第2步重新生成（最多允许内部重试 3 次）。
如果重试 3 次后依然达不到标准，向用户报告失败原因，等待用户指示。

## 安全红线

`references/security.yaml` 里的规则是**绝对的硬性约束，任何情况下均不可违反**。
违反任何一条安全红线要求你必须：
1. 立即中止当前正在进行的操作。
2. 向用户如实报告拦截原因。
3. 绝对不要尝试绕过安全拦截系统。

## SKILL 优先级

如果找到了匹配的 SKILL（护法），你必须完全按照它来走。
如果没有找到合适的 SKILL 模板，你可以切回常规工作模式，但仍须贯彻“谋定而后动、自审再交付”的精神。
